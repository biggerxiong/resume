麻将牌库服务2021.11 - 2021.12

通过控制玩家初始手牌，管理人员可在后台配置金币场玩家胡各种牌的概率、胡牌步长、杠牌的个数等，达到提升玩家体验、加快游戏进程的目的。
子游戏每次分牌前都访问本服务，本服务根据后台配置，通过相应算法计算出合适的牌后发给子游戏。

  • 从设计到编码都由本人完成
  • 使用 Grpc 进行服务间通信，供有需要的游戏调用
  • 使用 Gin 搭建 HTTP 服务器，供网页配置后台使用
  • 使用 Redis 进行游戏配置的缓存
  • 使用 Postgre SQL 和 Gorm 作为数据库和中间件
麻将机器人2020.12 - 2021.05

主导麻将机器人从 1.0 到 2.0 版本的开发。整合已有方案，使用向听数（离麻将胡牌还需多少张好牌）来作为当前手牌好坏的依据， 配合深度优先搜索算法来计算当前出牌最优解。
完成后，机器人胜率从 42% 提升到 53%～57%，极端情况下出牌速度较之前提升 70%。 1.0 版本占用太多服务器资源，导致经常收到服务器报警，2.0 版本后再未收到报警。
至今，已有超过 60 款麻将使用了此套机器人，日均对局 60W+。

  • 主导整个项目，机器人代码全由本人完成
  • 实现了一种可以将 14 张手牌压缩进一个 64 位数字的算法。麻将 14 张手牌极端情况下有 10^14 种可能， 算法将其可能压缩到 170 万种，基于此算法，麻将机器人内部计算向听数比之前快了 6 倍。
内部算法库优化2021.07 - 2022.02

内部算法库初次编写时间在 2017 年，后续很少再有人对其进行改动。 本项目对其中一些函数运行效率进行了优化，优化后判断胡牌算法速度提升 80% ，部分算法 30% ～ 70% 的提升。

  • 优化的方式有： 1. 优化算法逻辑，如 O(n^2) 优化到 O(n)； 2. 避免多余的内存分配； 3. 使用 CPU Profiling 定位耗时操作，针对性优化
  • 实现了一种快速检测麻将带精胡牌的算法，该算法将胡牌情况压缩进一个 32 位整数中， 所有整数放在数组中，检测时在数组上进行二分查找，优化后判断胡牌算法速度提升 80%。
